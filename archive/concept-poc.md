# Motive Layer — コンセプト定義書

## 概要

既存AIエージェント（Claude Code, Devin, OpenClaw等）に「動機」を与えるフレームワーク/レイヤー。
知能を作るのではなく、既にある知能に「動く理由」を与える。

- 新規のAIエージェントではなく、既存エージェントに取り込んで強化するフレームワーク
- コーディングだけでなく、あらゆる作業に使う汎用ツール
- 類似の取り組みは現時点で存在しない（AutoGPT/BabyAGIは再帰分解のみで発散して失敗、OpenClawは反応型、Claude Code/Devinは指示待ち、Linear/Asana等のAIはタスク管理補助のみ）

## 解決する課題

現状のAIエージェントは「指示がないと動けない」。人間がタスクを与えて初めて動く。
人間は「締切とゴール」を起点に、逆算して今やるべきことを判断し、行き詰まったら別タスクに切り替えている。
この動機構造をエージェントに与える。

## コアメカニズム

### メタ動機レイヤー：好奇心

4要素モデルの上位に位置する。3類型の動機（締切・不満・機会）が「既存のゴールをこなす」層であるのに対し、好奇心は**ゴール自体を生成・更新する**動機。

発動条件（例）:
- タスクキューが空いた時
- 想定外の結果を観測した時
- 前回失敗した領域がある時

**学習フィードバック（好奇心の方向づけ）**: タスク完了後に「何がうまくいった/いかなかった」を蓄積し、次のゴール生成に反映する。「前回失敗した領域を探索する」「成功パターンを他のゴールに転用する」といった行動がここから生まれる。

具体的な仕組みとして、**「状態 → 行動 → 結果」のログ**を記録する。state vectorで実行前の状態を記録し、選択した行動を記録し、実行後のstate vectorの変化を結果として記録する。このログにより「どの状態でどの行動が成功/失敗したか」のパターン分析が可能になる。また好奇心の方向づけにも活用できる：過去にstate vectorが大きく変化した領域（＝効果の高かった行動が存在する領域）を優先的に探索対象とすることで、好奇心を根拠のある方向に絞り込める。

### 4要素モデル

1. **ゴール** — あるべき/ありたい状態（持続的に保持）。**達成閾値**を持ち、ギャップがその閾値以下になった時点で完了と判断する（詳細は「十分の判断」セクションを参照）
2. **現在地** — 今どこまで進んでいるか（観測ベース + 確信度）
   - 自己申告ではなく、成果物の観測から機械的に導出する
   - 単一の値ではなく「観測値 + 確信度」のペアで保持
   - 観測値の内部表現として**状態ベクトル（state vector）**を導入する。状態ベクトルとは、現在の状況を数値化したベクトル（例：進捗率、品質スコア、未解決課題数、リソース消費率など）。各要素に確信度を持たせることで「観測値 + 確信度」モデルと整合する
   - これによりゴールとのギャップが機械的に計算可能になり、「なぜその行動を選んだか」が観測可能な形で透明になる
3. **ギャップ** — ゴールと現在地の差分から「次にやること」を動的に生成
4. **制約** — タスクのフィルタリング・破棄（3軸）
   - 発散防止（量の制御）: 世代制限、タスクバジェット、距離フィルタ
   - 不確実性制約（質の制御）: 確信度が低い→行動範囲を狭める/人間に確認
   - リソース制約: コンテキスト残量・API呼び出し残量・予算・時間に応じてタスクの重さを調整

```
【メタ動機レイヤー】
  好奇心（ゴールを生成・更新）← 学習フィードバック（完了結果の蓄積）
        │
        ▼
  ゴール（達成閾値付き）
        │
  現在地（観測ベース + 確信度）
        │
       ギャップ
        │
       制約（3軸）
        ├─ 発散防止（量の制御）
        ├─ 不確実性制約（質の制御）
        └─ リソース制約
        │
       タスク
```

### 動機の3類型（動的スコア）

優先度を固定テーブルではなく、文脈・時間経過で変動する動的スコアで管理する。タスク選択はスコアが最も高い動機から行う。state vectorの各要素（進捗率・品質スコア等）からスコアを算出することで、スコア変動の根拠が透明になる。

| 類型 | 説明 | タスク変換方式 | スコア変動の特性 |
|------|------|----------------|-----------------|
| 締切駆動 | いつまでにこうなっていなければならない | 逆算型 | 締切が遠い→低、近づくと急上昇 |
| 不満駆動 | 今の状態がゴールに照らして不十分だから直す | 差分検出型 | 放置・慣れで減衰 |
| 機会駆動 | 今やると効率が良い、後で楽になる | イベント駆動型 | 鮮度で減衰 |

## 付随メカニズム

### 十分の判断（satisficing）

人間は完璧を目指さずsatisficing（満足化）する。エージェントは放置すると際限なく改善し続けがち。

ゴールに**達成閾値**を持たせ、ギャップが閾値以下になった時点を完了とする。確信度との組み合わせ:
- 確信度が高い × ギャップが閾値以下 → 完了
- 確信度が低い × ギャップが閾値以下 → 完了宣言前に追加検証タスクを生成

### 停滞検知→切り替え

進捗停滞を検知→ギャップの再計算→別タスクの選択。

検知指標:
- 見積もり時間の2倍経過
- 同種アクション3回連続失敗
- 出力の変化率低下

エスカレーション分岐:
- 情報不足 → 調査タスクを自己生成
- 権限不足 → 人間にエスカレーション
- 能力不足 → ゴールの再定義を人間に要請
- 外部依存 → 待機 + 別タスクに切り替え

### 人間との協調（信頼残高 × 確信度）

完全自律はゴールではない。行動の自律度を2つの軸で連続的に調整する。

- **信頼残高**（外部制御）: 人間がどれだけエージェントに任せるか。成功で回復、失敗で減少。ゴール単位で付与。
- **確信度**（内部制御）: エージェント自身がどれだけ確かか。観測ベースの現在地は高、自己推定は低。

| 信頼残高 | 確信度 | 振る舞い |
|----------|--------|----------|
| 高 | 高 | 自律実行 |
| 高 | 低 | タスク生成するが人間に確認 |
| 低 | 高 | タスク生成するが人間に確認 |
| 低 | 低 | 現在地確認タスクを先に生成 |

不可逆アクション（外部API呼び出し、デプロイ等）は信頼残高・確信度に関わらず常に人間承認。

既存エージェントが「自信満々で間違える」か「常に人間に聞く」の二択なのに対し、Motive Layerは確信度に応じて自律度を連続的に調整する（差別化ポイント）。

## AGIとの違い

知能そのものではなく、知能を駆動する動機構造を提供する。
既にある知能（LLM）に「動く理由」を与えるレイヤーであり、知能の向上は目的としない。
